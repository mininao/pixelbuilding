Présentation du projet

L'idée de départ de ce projet est d'utiliser les salles de classes d'un épi en tant que pixels d'un écran.
Pour cela, la solution la plus simple était d'utiliser les vidéo projecteurs déjà présents dans les salles, et de les tourner vers le mur opposé à la fenêtre.
Côté logiciel, le plus simple était là encore d'utiliser un navigateur web dans chaque salle, nous permettant ainsi de nous passer de client lourd.

Démonstration

Pour tester, le projet, vous pouvez vous rendre à l'adresse communiquée. l'url /pixel/test permet de simuler le comportement des pixels (via des iframes) l'url /pixel/dashboard vous donne accès au panneau de contrôle

Notez que cette version est (très) lente, puisqu'il s'agit en réalité d'un VPN vers un serveur pour utilisation locale à l'ESIEE

Fonctionnalités

Gestion d'un tableau de pixels

Chaque "pixel" possède un identifiant unique, des coordonnées, une couleur et un nom d'affichage. Le stockage, l'accès et la modification des pixels sont gérés par le module core.js

API HTTP

PixelBuilding fournit des API simples en HTTP (gérées par http_api.js) Vous pouvez par exemple essayer les requêtes suivantes :

  • GET /pixel/http_api/screens Pour obtenir une liste json de tous les pixels
  • GET /pixel/http_api/coordinates/X/Y Pour obtenir les informations d'un pixel particulier
  • POST /pixel/http_api/coordinates/X/Y?color=green Pour changer la couleur d'un pixel

Temps Réél avec les websockets

Pour accélérer la transmission des changements de couleur aux pixels, j'utilise la librairie socket.io qui permet une communication bilatérale client-serveur, et par conséquent de s'affranchir de la structure Requête-Réponse.

Dashboard simple et élégant

Afin de permettre à l'utilisateur final d'utiliser PixelBuilding simplement, il a été nécessaire de créer un panneau de contrôle adapté. Celui-ci est conçu à l'aide de la partie "grille" du framework foundation et en scss Ce panneau est toujours en construction et certaines de ses fonctionnalités ne sont pas disponibles
